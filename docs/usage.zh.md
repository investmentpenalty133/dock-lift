使用指南
===

## 安装与首次启动

1. 从 Mac App Store（或你的分发安装包）安装 DockLift。  
2. 打开应用 — 菜单栏出现图标（默认不在 Dock 占位）。  
3. 按系统提示在 **系统设置** 中授予 **辅助功能**。  
4. 建议保持 **启用 DockLift** 打开。

## 日常使用

### 从其它 Space 拉回窗口

1. 在 Space A 打开某个应用窗口。  
2. 切换到 Space B。  
3. 在 Dock 点击该应用图标。  
4. DockLift 会尽量在 Space B 显示最近使用的窗口。

### 从其它显示器拉回窗口

1. 将应用窗口移到外接显示器。  
2. 把指针移到你工作的显示器（点击 Dock 的那块屏）。  
3. 在 Dock 点击该应用。  
4. 即使应用在外接屏上已是前台，只要开启了 **将窗口移到 Dock 所在屏幕**，DockLift 也应把窗口拉过来。

## 设置说明

从菜单栏打开 **设置…**（`⌘,`）。

### 通用

| 选项 | 含义 |
|------|------|
| 启用 DockLift | 总开关 |
| 仅响应 Dock 点击 | 关闭后，⌘Tab 等激活也会处理 |
| 包含最小化窗口 | 可恢复最小化窗口 |
| 将窗口移到 Dock 所在屏幕 | 多显示器重新定位 |
| 将窗口移到当前空间 | Space 移动（尽力而为） |
| 必要时使用最小化回退 | Space 的公开 API 回退 |
| 登录时启动 | 随用户登录启动 |
| 在菜单栏图标旁显示标题 | 可选显示 “DockLift” 文字 |

### 权限

查看辅助功能状态；如未授权，可在系统设置中开启。

### 高级 — 忽略列表

列表中的应用不会被提升。添加方式：

- 输入 **Bundle 标识符** 后点 **添加**  
- 将 **.app 拖到虚线区域**  
- **选择应用…**（文件面板）  
- **正在运行的应用** 菜单  

默认忽略访达与 DockLift 自身（列表可改）。

## 故障排除

| 现象 | 可尝试 |
|------|--------|
| 点 Dock 没反应 | 确认已授辅助功能；已启用 DockLift；应用不在忽略列表 |
| 应用已是前台但窗口仍在外接屏 | 使用最新版本；打开“移到 Dock 所在屏幕”；再点一次 Dock 图标 |
| 某些应用无法移动 | 全屏 / 部分 Electron 应用可能忽略 AX 位置；可先退出全屏 |
| 功能无反应 | 在 **系统设置 → 隐私与安全性 → 辅助功能** 中授予权限 |
| 拖拽加入忽略列表失败 | 改用“选择应用…”或“正在运行的应用”；确保拖入的是 `.app` |
